\subsection{Detailed Description}
\subsubsection*{template$<$typename TYPE$>$ class Cfg\_\-Int$<$ TYPE $>$}

Cfg class that takes care of integer types. This template is needed since wxValidator only supports normals ints, and wxConfig for the matter only supports longs, thus some worksarounds are needed.

There are two work-\/arounds: 1) wxValidator only supports int$\ast$, so we need a immediate variable to act as a storage. Thus we use \doxyref{Cfg\_\-Tmpl$<$int$>$}{p.}{classCfg__Tmpl} as base class. Thus this class contains a integer which we use to pass the value back and forth between the widgets.

2) wxConfig uses longs to save and read values, thus we need an immediate stage when loading and saving the value.
