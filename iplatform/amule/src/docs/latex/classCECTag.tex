\subsubsection[{AddTag}]{\setlength{\rightskip}{0pt plus 5cm}bool CECTag::AddTag (const {\bf CECTag} \& {\em tag}, \/  {\bf CValueMap} $\ast$ {\em valuemap} = {\ttfamily NULL})}\label{classCECTag_ad2a4413128ac2b3097e44a60836276a3}


Add a child tag to this one. The tag argument is reset to an empty tag.

Be very careful that this method swallows the content of {\itshape tag\/}, leaving {\itshape tag\/} empty. Thus, the following code won't work as expected: 
\begin{DoxyCode}
 {
    CECPacket *p = new CECPacket(whatever);
    CECTag *t1 = new CECTag(whatever);
    CECTag *t2 = new CECTag(whatever);
    p->AddTag(*t1);
    t1->AddTag(*t2);    // t2 won't be part of p !!!
 }
\end{DoxyCode}


To get the desired results, the above should be replaced with something like:


\begin{DoxyCode}
 {
    CECPacket *p = new CECPacket(whatever);
    CECTag *t1 = new CECTag(whatever);
    CECTag *t2 = new CECTag(whatever);
    t1->AddTag(*t2);
    delete t2;  // we can safely delete the now empty t2 here, because t1 holds i
      ts content
    p->AddTag(*t1);
    delete t1;  // now p holds the content of both t1 and t2
 }
\end{DoxyCode}


Then why copying? The answer is to enable simplifying the code like this:


\begin{DoxyCode}
 {
    CECPacket *p = new CECPacket(whatever);
    CECTag t1(whatever);
    t1.AddTag(CECTag(whatever));    // t2 is now created on-the-fly
    p->AddTag(t1);  // now p holds a copy of both t1 and t2
 }
\end{DoxyCode}
