\subsection{Detailed Description}
\subsubsection*{template$<$typename ValueType$>$ class CObservableQueue$<$ ValueType $>$}

This class forms the superclass for observable queues or lists. The protocol defined above is used (\doxyref{CQueueEvent}{p.}{classCQueueEvent}).

By subclassing this class, a container object can ensure that classes observing it are able to keep in sync with the changes made to the contents, with a few limits.

For one thing, the value is assumed to be both key and value, so multimaps cannot be used with this class, since it will only pass the key. Nor can lists that rely on a specific order be used, since neither insertion nor deletion operations specify a specific object or position, instead just a \char`\"{}add/remove one of these\char`\"{} rule.

The main purpose of this class is to allow another class to follow a list or queue, regardles of changes in actual order of the items and regardles of changes to the contents.
